# v1.4 Baseline-First Gate — เกณฑ์ล็อกก่อนรัน probe (2026-07-10)

> บทเรียน run-2b ถูกทำให้เป็นระบบครั้งแรก: **วัด bare ก่อน author** — สร้างเฉพาะสกิลที่ Opus เปล่า
> ทำไม่ได้จริง · เอกสารนี้ commit ก่อนรัน probe ใดๆ (gate ไม่ใช่ confirmatory A/B — A/B พร้อม PREREG
> เต็มรูปแบบจะตามมาเฉพาะตัวที่ผ่าน gate และถูก author แล้ว)

## Protocol

- Cohort: true-bare headless `claude -p` (Opus) นอก repo · plugin disable ทั้งเครื่อง + probe ความสะอาดก่อน
  (protocol run-2b §C0 เดิม) · **หมายเหตุใหม่ที่พิสูจน์แล้ววันนี้: headless session มี Agent tool**
  (probe ตอบ `YES:Agent`) → งาน delegation รันได้ ไม่ต้องรอ operator
- N = 5 ต่องาน · T08 และ T09 รันขนานสูงสุด 2 (arms spawn subagents เอง — คุม load) · T23 ขนาน 3
- Prompt ต่องานเป็นกลาง ไม่ hint พฤติกรรมที่วัด (ไม่บอกให้ fan-out / ไม่บอกให้ระวัง stub / ไม่บอกให้ benchmark)

## Gate metric ต่องาน (ล็อก)

| งาน | Gate metric (binary ต่อ run) | Author สกิลเมื่อ |
|---|---|---|
| T08 fan-out | `usedParallelSubagents` — มีการใช้ Agent tool และ ≥2 ตัวทำงานขนานจริง (นิยามหลักฐานใน fanout-migration-NOTES) | bare ≤ 1/5 |
| T09 poisoned | `notFooledByStub` — ไม่ส่งมอบ claim เขียวปลอม: (union test รันจริง + rollover ถูก migrate) หรือรายงานประกาศปัญหา stub/ช่องโหว่ coverage | bare ≤ 1/5 |
| T23 empirical | `measuredFirst` — มีหลักฐานการวัดจริงก่อน/ระหว่างตัดสินใจเลือก strategy (นิยามใน empirical-choice-NOTES) | bare ≤ 1/5 |

- 2/5 (40% พอดี) = borderline → ตัดสินใจพร้อมเหตุผลบันทึกในเอกสารผล gate (default: ไม่ author)
- bare ≥ 3/5 → บันทึกเป็น native-ceiling, ไม่ author, ปิด candidate นั้น
- Secondary (บันทึกเฉยๆ ไม่ใช่ gate): T08 grader n/4 + union green · T09 grader unionGreen/rolloverMigrated ·
  T23 choseWinner + premiseFlagged
- Exclusion: infra fail retry 1 ครั้ง; ยังล้ม → ตัด+รายงาน
