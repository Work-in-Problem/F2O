FABLE2OPUS — USER GUIDE

คู่มือการใช้งาน Fable2Opus

สอน Claude Opus 4.8 ให้ทำงานด้วย "กระบวนการ" ระดับ Fable 5 — ติดตั้งครั้งเดียว ใช้ได้ทุกโปรเจกต์
เวอร์ชัน 1.3 · กรกฎาคม 2026ติดตั้งด้วยคำสั่งเดียวไม่ต้องมีพื้นฐานลึก

1โปรเจกต์นี้คืออะไร

Fable2Opus (F2O) คือชุด "ถ่ายทอดกระบวนการทำงาน" — เกิดจากการวิเคราะห์ว่า Claude Fable 5 (รุ่นท็อป) ทำงานต่างจาก Claude Opus 4.8 ตรงไหน แล้วกลั่นความต่างส่วนที่เป็นวินัยและขั้นตอน (ไม่ใช่ความฉลาดดิบ) ออกมาเป็นกติกา+คู่มือที่ Opus 4.8 ทำตามได้จริง ผลคือได้ Opus ที่ทำงานรอบคอบระดับใกล้เคียงรุ่นท็อป ในราคาถูกกว่าครึ่งหนึ่งและ limit การใช้งานที่เยอะกว่า

ทำอะไรได้บ้าง

  • งานโค้ดทุกชนิด — แก้บั๊ก สร้าง feature, refactor, migration แบบมีหลักฐานทุกขั้น
  • ล่าบั๊กยาก — บั๊กเป็นๆ หายๆ, race condition ด้วยวิธีเชิงสถิติ ไม่ใช่เดา
  • Code review — หาให้ครบก่อนกรอง ไม่ปล่อยบั๊กหลุดเพราะถูกสั่งให้ "เอาแต่เรื่องใหญ่"
  • งานยาวข้ามวัน/ข้ามsession — จดโน้ตกันงานหล่นเมื่อ context เต็ม
  • งานเอกสาร — รายงาน spreadsheet slide ที่ตัวเลขตรวจทานแล้วทุกตัว
  • อ่านค่าจากภาพ/สแกน — ตาราง ตัวเลข แบบทวนสอบ ไม่เดาจุดที่มองไม่ชัด
  • งานใหญ่แบบขนาน — แตกงานให้ sub-agent หลายตัวช่วยแล้วตรวจรวม
  • กันหลอกว่าเสร็จ — hook บล็อกคำว่า "done" ที่ไม่มีหลักฐานจริง

2ใช้ F2O กับไม่ใช้ ต่างกันอย่างไร

สถานการณ์Opus 4.8 เพียวๆ (พฤติกรรมตั้งต้นที่ Anthropic บันทึกไว้)Opus 4.8 + F2O
บั๊กเป็นๆ หายๆ✗ เทสต์ผ่าน 1 รอบก็ประกาศ "fixed"✓ บังคับรันซ้ำ 10–30 รอบ รายงานตัวเลขก่อน/หลังแก้
คำว่า "เสร็จแล้ว"✗ ต้องเชื่อคำพูด ไม่รู้ตรวจจริงไหม✓ ทุก claim ชี้หลักฐานได้ · แยก VERIFIED / NOT VERIFIED ชัด · hook บล็อกถ้าโกหก
คำถามระหว่างงาน✗ ถามเรื่องเล็กบ่อย จบเทิร์นด้วย "เอาไหมครับ?"✓ เรื่องย้อนกลับได้ตัดสินใจเอง+จดไว้ให้ veto ทีหลัง — งานจบในเทิร์นเดียวบ่อยขึ้น
ข้อเท็จจริง เวอร์ชัน/API/ราคา✗ ตอบจากความจำ (อาจล้าสมัย)✓ ค้นเว็บ/อ่านไฟล์จริงก่อนพูดเสมอ
ขอบเขตการแก้โค้ด✗ ชอบ refactor แถม diff บวม✓ diff ตรงโจทย์ ปัญหาข้างเคียงถูก "จดรายงาน" แทนแก้มั่ว
Review ที่สั่งว่า "เอาเฉพาะเรื่องใหญ่"✗ ทำตามตรงตัวจนตกหล่นบั๊ก (recall ร่วง — Anthropic วัดเอง)✓ หาให้หมดก่อน แล้วค่อยกรองตอนรายงาน
งานหลาย requirement✗ ข้อท้ายๆ ชอบหล่นหาย✓ checklist ครบทุกข้อ ปิดงานเทียบหลักฐานรายข้อ (n/n)
งานยาว / ทำต่อวันหลัง✗ ลืมบริบท ทำซ้ำหรือหล่นหลัง context เต็ม✓ โน้ต WORKING_NOTES.md + พิธีฟื้นบริบทจากหลักฐานจริง
เพื่อความตรงไปตรงมา: คอลัมน์ซ้ายคือพฤติกรรมตั้งต้นของ Opus 4.8 ตามเอกสาร migration ของ Anthropic เอง ส่วนขวาคือกลไกที่ F2O บังคับ · หลักฐานเชิงตัวเลขที่มีแล้ว: instruction แบบเดียวกันนี้ลดการถามพร่ำเพรื่อ ~12 จุด (Anthropic วัด) และการทดสอบ A/B ของโปรเจกต์พบการรายงานปัญหาข้างเคียงเพิ่มจาก 1/5 → 5/5 เมื่อโหลด skill · สิ่งที่ F2O ให้ไม่ได้: ความฉลาดดิบ — ช่องว่างด้าน insight ยังเป็นของรุ่นท็อป

3ส่วนประกอบ 3 ชั้น

ชั้นคืออะไรจำเป็นไหม
Layer 0กฎแกน 2 ไฟล์ (CLAUDE.md + constants.md) — ห้ามบอกว่า "เสร็จ" โดยไม่มีหลักฐาน · ไม่ถามเรื่องเล็กน้อย · จดโน้ตไฟล์เดียวจำเป็น
Layer 1Skills 13 ตัว — คู่มือเชิงลึกที่ Claude โหลดเองอัตโนมัติตามสถานการณ์ (แก้บั๊ก → ตัว debug ทำงาน, จะสรุปงาน → ตัวรายงานทำงาน)จำเป็น
Layer 2Hook ตรวจจับ — สคริปต์ที่บล็อกข้อความ "เสร็จแล้ว" ที่ไม่มีหลักฐานจริง ก่อนถึงมือคุณทางเลือก (แนะนำ)
สิ่งที่จะเปลี่ยน
  • ค้น/อ่านโค้ดก่อนแก้ ไม่เดาจากความจำ
  • รันเทสต์ซ้ำก่อนพูดว่า "fixed" (บั๊กแบบเป็นๆ หายๆ รันซ้ำ 10–20 รอบ)
  • ทำงานจนจบเทิร์น ไม่ถาม "เอาไหมครับ?" กลางทาง
  • สรุปงานแบบ VERIFIED / NOT VERIFIED / ASSUMED
สิ่งที่ไม่เปลี่ยน

ความฉลาดดิบของโมเดล — ชุดนี้ถ่ายทอดวินัยการทำงาน ไม่ใช่สมองเสริม งานที่ต้องใช้ insight ลึกยังเป็นขีดจำกัดของตัวโมเดลเช่นเดิม

4สิ่งที่ต้องมีก่อน

  • Claude Code (แอป desktop หรือ CLI) ที่ล็อกอินใช้งานได้แล้ว
  • โมเดล Opus 4.8 — ใน session พิมพ์ /model แล้วเลือก Opus (ชุดนี้ออกแบบเพื่อ Opus 4.8 แต่ใช้กับโมเดลอื่นก็ไม่พัง)
  • Python 3 — เฉพาะถ้าติดตั้ง Layer 2 (macOS มีอยู่แล้ว — เช็ค: python3 --version)
แหล่งข้อมูล: GitHub github.com/Work-in-Problem/F2O (open source, MIT) · เว็บไซต์ f2o.workinproblem.com — ติดตั้งด้วยคำสั่งเดียวได้ 2 ช่องทาง (ดูหัวข้อ 5) หรือ clone มาติดตั้งมือ

5ติดตั้ง

ทาง A · แนะนำClaude Code plugin — ครบทุกชั้น: skills + กฎแกนฉีดอัตโนมัติ + hook
claude plugin marketplace add Work-in-Problem/F2O
claude plugin install f2o@f2o

หรือพิมพ์ /plugin ในแอปแล้วเลือก marketplace Work-in-Problem/F2O — เปิดใช้แล้วกฎ Layer 0 ถูกฉีดเข้าทุก session อัตโนมัติ · ควบคุม: /f2o:off (soft-off — สกิลยังใช้ได้) · /f2o:on · /f2o:status · shorthand: /f2o:vbc ฯลฯ ครบ 13 ตัว

ทาง Bnpx skills — ได้เฉพาะ skills 13 ตัว
npx skills add Work-in-Problem/F2O --all -y -g

ช่องทางนี้ไม่พกกฎแกน (Layer 0) กับ hook — copy เพิ่มเองตามทาง C ขั้น 2–3

ทาง C — ติดตั้งมือจาก clone (~2 นาที)

เปิดแอป Terminal แล้ว copy คำสั่งไปวางทีละกล่อง

ขั้น 1ติดตั้ง Skills 13 ตัว แบบ global — ทำครั้งเดียว ใช้ได้ทุกโปรเจกต์
git clone https://github.com/Work-in-Problem/F2O.git && SRC="$PWD/F2O"

mkdir -p ~/.claude/skills
cp -R "$SRC/skills/." ~/.claude/skills/

# ตรวจว่าเข้าครบ — ต้องเห็น 13 โฟลเดอร์
ls ~/.claude/skills
ขั้น 2ติดตั้ง Layer 0 ลง "โปรเจกต์ที่จะใช้งาน" — ทำซ้ำกับทุกโปรเจกต์ที่ต้องการ
cd /path/to/your-project        # ← แก้เป็นโปรเจกต์จริงของคุณ

cp "$SRC/core/constants.md" ./constants.md

# ถ้าโปรเจกต์ยังไม่มีไฟล์ CLAUDE.md:
cp "$SRC/core/CLAUDE-core.md" ./CLAUDE.md

# ถ้ามี CLAUDE.md อยู่แล้ว ให้ต่อท้ายแทน:
cat "$SRC/core/CLAUDE-core.md" >> ./CLAUDE.md
ขั้น 3 · ทางเลือกติดตั้ง Hook กันหลอกว่าเสร็จ (Layer 2)
mkdir -p .claude/hooks
cp "$SRC/hooks/claim_gate.py" .claude/hooks/
cat > .claude/settings.json <<'JSON'
{"hooks":{"Stop":[{"hooks":[{"type":"command","command":"python3",
  "args":["${CLAUDE_PROJECT_DIR}/.claude/hooks/claim_gate.py"],"timeout":30}]}]}}
JSON
ระวัง: ถ้าโปรเจกต์มี .claude/settings.json อยู่แล้ว อย่า copy ทับ — เปิดไฟล์แล้วเพิ่มเฉพาะก้อน "hooks" จากไฟล์ต้นทางเข้าไปแทน

6เรียกใช้ยังไง

คำตอบสั้น: ไม่ต้องเรียก — มันทำงานเอง
  1. cd เข้าโปรเจกต์ที่ติดตั้งแล้ว → เปิด claude → พิมพ์ /model เลือก Opus 4.8
  2. สั่งงานตามปกติ — Claude โหลด skill ที่ตรงสถานการณ์เองอัตโนมัติ
คุณสั่งว่า…Skill ที่ทำงานเอง
"แก้บั๊ก login แล้วเทสต์ให้ผ่าน"root-causing-bugs + verifying-before-claiming
"รีวิว PR นี้หน่อย"reviewing-code
"ทำรายงานยอดขายจากไฟล์ csv นี้"producing-deliverables

อยากเรียกเองเจาะจงก็ได้: พิมพ์ / แล้วเลือกชื่อ skill หรือบอกตรงๆ ว่า "ใช้ skill scoping-code-changes ด้วย"

เคล็ดสำคัญกับ Opus 4.8: ให้ โจทย์ครบตั้งแต่ข้อความแรก — Opus ทำงานยาวได้ดีที่สุดเมื่อได้ spec เต็มในเทิร์นเดียว อย่าป้อนทีละนิด แล้วปล่อยให้ทำจนจบ ไม่ต้องจ้ำจี้ระหว่างทาง

7รู้ว่าติดตั้งสำเร็จจากอะไร

เปิด session ใหม่แล้วลองสั่ง: "แก้ typo ในไฟล์ X แล้วบอกด้วยว่า verify ยังไง" — สัญญาณที่ควรเห็น:

  • อ่านไฟล์ก่อนแก้ และไม่ทำพิธีใหญ่โตกับงานจิ๋ว (กติกา proportionality ทำงาน)
  • สรุปปิดงานขึ้นต้นด้วยผลลัพธ์ 1 บรรทัด + หมวด VERIFIED พร้อมคำสั่งที่รันจริง
  • ไม่จบเทิร์นด้วย "Want me to also…?" สำหรับสิ่งที่ควรทำต่อเอง
  • เช็คแบบกลไก (ทาง B/C): ls ~/.claude/skills | wc -l ต้องได้ 13 · ทาง A (plugin): claude plugin list ต้องเห็น f2o enabled

8ปัญหาพบบ่อย

อาการทางแก้
Skill ไม่โหลด / พิมพ์ / แล้วไม่เห็นปิด-เปิด session ใหม่ (skills โหลดตอนเริ่ม session) · ตรวจ ls ~/.claude/skills
Hook บล็อกทั้งที่ไม่ควรบล็อกปิดชั่วคราว: export FABLE2OPUS_HOOKS_OFF=1 แล้วเปิด session ใหม่
ทำพิธีเยอะเกินไปกับงานเล็กบอกตรงๆ ได้เลย ("งานนี้เล็ก ข้ามพิธี") — ทุกกฎมีช่องยกเว้นพร้อมเหตุผล 1 บรรทัดอยู่แล้ว
ใช้ token มากขึ้นปกติ — งาน verify หนักใช้ ~1.3–2 เท่า แลกกับความน่าเชื่อถือ งานเล็กแทบไม่ต่าง
เลขในกฎขัดกันเองconstants.md ชนะเสมอ (กติกา Layer 0 ข้อ 4)

9อัพเดต / ถอนการติดตั้ง

ต้องการวิธีทำ
อัพเดตแก้ที่โฟลเดอร์ต้นทาง แล้วรันคำสั่ง copy ขั้น 1–3 ซ้ำ (ทับของเก่า)
ถอน skillsลบโฟลเดอร์ที่ไม่ต้องการออกจาก ~/.claude/skills/ (เลือกลบรายตัวได้)
ถอน Layer 0ลบ constants.md และส่วน Fable2Opus ใน CLAUDE.md ของโปรเจกต์
ถอน hookลบก้อน hooks ใน .claude/settings.json + ลบโฟลเดอร์ .claude/hooks/
